# Issue Notes

This folder stores RAG-friendly notes for real OpenClaw issues found in the
field.

The goal is not just "write docs." The goal is to keep a retrievable operational
memory that can be reused for:

- future debugging
- support replies
- Facebook posts
- Reddit posts
- issue reports and fix notes

## Format

Each issue note should keep the same sections:

1. `Summary`
2. `Symptoms`
3. `Root cause`
4. `Why it happens`
5. `Fix`
6. `How to verify`
7. `Short post draft`

That structure is simple enough for retrieval and strong enough for
community-facing writeups.
